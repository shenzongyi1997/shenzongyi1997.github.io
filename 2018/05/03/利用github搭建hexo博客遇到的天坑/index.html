利用github搭建hexo博客遇到的天坑

好玩吗

我靠，一点也不好玩啊，都要吐血了

今天因为原来的阿里云服务器上被我用来放了一个做互联网+需要用的网页，重新清了一遍，装上了Tomcat和mysql等组件，把原来的wordpress给删除了。所以原来的博客就不用啦，不过本来那博客好像只是用来装x而并没啥干货。。。。

所以利用最方便的github搭建了自己的第二个博客。

自己想认认真真学点东西，认真写一点文章，因为一直处于读什么都感兴趣对什么都浅尝辄止的状态，所以一直有一些迷茫。之前阿里面试挂了之后发现自己好像真的没什么特别能拿得出手的项目，而且投简历的时候我都不太清楚自己想投什么岗，好像java研发岗是最适合我的岗位了，不过还是很菜orz

今天搭建这个博客的时候，遇到的天坑就是config.yml文件里修改repo的时候，冒号后一定要加空格！！！做了好几遍，发现本地运行完全没问题，但是就是deploy不到github上，密钥用户都设置对了，但是那个repo里还是万年不动的两个文件，一个index一个config.yml，当时简直怀疑人生啊。

恩。。。第一篇博文就先写这么多，下一篇估计是ESPRIT算法，最近在学习的东西。自己选择了一个很有意思的导师，在学习通信算法领域的知识，发现好多数学知识自己都已经有些生疏了，线性代数和矩阵运算在里面层出不穷，最让我不太能理解的就是复数的出现， 虽然我知道物理世界里信号都是是信号，I只是用来表示的一种方法，但我就是死活理解不了为什么要用i。我一直喜欢用感性的方式去理解那些理性的东西，这可能也是我学习效率比较低的原因之一把orz
